认识JavaScript-JavaScript是什么

JavaScript一种直译式脚本语言，是一种动态类型、弱类型、基于原型的语言，内置支持类型。它的解释器被称为JavaScript引擎，为浏览器的一部分，广泛用于客户端的脚本语言，最早是在HTML（标准通用标记语言下的一个应用）网页上使用，用来给HTML网页增加动态功能。—-来自百度百科

JavaScript具有的特点

我们通过上面的一段话，可以看出JavaScript具有以下特点：

  • 脚本语言
  • 具有动态类型的特点
  • 具有弱类型的特点
  • 它是基于原型的一种语言
  • 它需要解释器来执行-可以理解为需要浏览器
  • 它是一种客户端的脚本语言

这是我们通过上面百度百科的解释来总结出来的它具有的一些特点。

JavaScript的组成部分

它由三大部分组成：ECMAScript基本语法和库，DOM对象，BOM对象；

  • ECMAScript：描述了他的基本语法要求和一些基本的内置库的规范支持
  • DOM对象：Document Object Model 文档对象模型
  • BOM对象：Browser Object Model 浏览器对象模型

总结一句话就是：ECMA是规范和语法约束，DOM和BOM是API接口操作，DOM是操作HTML等标签的，BOM是操作浏览器相关接口的API。

JavaScript中的数据类型

内置的六种基本数据类型

  • Number 数字类型
  • String 字符串类型
  • boolean 布尔类型
  • null 空类型
  • undefined 未定义类型
  • Object 引用对象类型

  1. Number 数字类型

    数字类型有一个取值范围是从Number.MAX_VALUE - Number.MIN_VALUE 的值这个是根据计算机的位数有关。

    在js中可以表示八进制和十六进制：

    • 八进制以0开头 例如 var number = 012; // 十进制的10

    • 十六进制以0x开头 例如 var number = 0x10; // 十进制的10

      注意：如果一个数类型与undefined类型进行运算或者与一个不能转换为数字类型的字符串类型进行非相加运算时，则会返回NaN的
      验证一个数据是不是NaN 使用isNaN()方法。 例如：

  2. String 字符串类型

    String 类型的独特之处在于，它是唯一没有固定大小的原始类型，字符串中每个字符都有特定的位置，首字符从位置 0 开始，第二个字符在位置 1，依此类推。

  3. undefined类型

    Undefined 类型只有一个值，即 undefined。当声明的变量未初始化时，该变量的默认值是 undefined。

  4. null 类型

    这个类型只有一个专用值，为null值。 undefined 实际上是从值 null 派生来的，因此 ECMAScript 把它们定义为相等的。但是他们的意义是完全是不一样的，null 则用于表示尚未存在的对象（在讨论 typeof 运算符时，简单地介绍过这一点）。如果函数或方法要返回的是对象，那么找不到该对象时，返回的通常是 null。

  5. 数据类型的转换

    字符串类型转换为数字类型：

    1. 利用parseInt()将字符串类型转换为整数
    2. 利用parseFloat()将字符串类型转换为浮点数
    3. 利用Number()将字符类型转换为数字类型

    注意：parseInt()、parseFloat(),转换的字符串必须以数字来开始，Number(),转换的字符串必须是严格的数字格式，否则他们的都将输出NaN

    数字类型转换为字符串类型

    1. 如果变量是有意义的就是用toString()的方法，将数字类型转换为字符串类型
    2. 如果没有意思可以利用String()的方法转换
    3. 隐式转换，就是利用‘+’字符串拼接的方法来进行转换

函数

函数的声明

函数的声明有一下几种方式：

  1. 直接命名的方式来声明：用function fnName(a){}
  2. 利用函数表达式的方式来声明： var abc = function (a){};

对于函数的命名有一下要求：获取信息的函数使用get开头例如，getAttribute();设置属性的函数使用set开头例如，setAttribute();判断的函数使用is开头，例如isElement();

这两种声明的区别主要在与函数的预解析问题，利用函数表示的方式，可以避免函数的预解析。

匿名函数：可以利用它来防止函数名冲突而导致的函数覆盖问，匿名函数在dom事件上，用于事件处理函数，同时匿名函数也经常用作回调函数，都是因为他具有防止函数冲突的优点。

作用域

作用域：就是变量的使用范围-分为局部变量作用域和全局变量作用域。

作用域链：就是变量的使用是从里向外层层的搜索，搜索到了就直接使用，一直搜索到全局的作用域上。

利用作用域链的原理和函数嵌套的关系，就可以形成闭包的效果。

闭包的模式：函数箱套关系，在一个函数内部调用另一个函数，只有内部的函数才能调用外部函数的内部变量，这就形成了一个闭包，简单理解就是，将一个函数包裹在另一个函数内。

闭包的作用：缓存数据延长作用域链-理解为可以利用闭包来实现，函数外面访问函数内部的变量。

闭包的缺点：会导致内存泄漏。

预解析的问题

预解析：提前解析代码，分为两个预解析一个是变量的声明，另一个是函数的声明。

变量声明的预解析：会将变量声明进行提前，但是赋值不会提前

注意：预解析只是将变量声明和函数声明提前，但是它只是提前到当前作用域的最上面，不会超出当前作用域。

跨越<script></script>的函数声明和变量声明不能进行预解析
